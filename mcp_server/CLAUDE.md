# mcp_server/ — MCP-сервер (stdio) для Claude Code

**Назначение.** Объявляет инструменты и роутит их через ядро. Это мост, через который
Claude Code управляет Jobber (сканирование, профиль, дайджест).

**Ключевые файлы.**
- `server.py` — точка входа stdio-сервера, регистрация инструментов.

**Инструменты (реализованы).**
- `sources()` — источники и capability-флаги.
- `get_profile()` — текст `cv.md`.
- `scan_candidates(depth_days?, include_archived)` — Telegram I/O + предфильтр → кандидаты (без LLM).
- `write_digest(items, date?)` — дедуп + рендер + запись датированного файла в Obsidian.
- `seen_reset()` — сброс курсоров/seen.

**Главный сценарий.** Сервер делает Telegram-I/O и запись; классификацию/скоринг/письма пишет САМ
Claude Code по `cv.md` (без API-ключа). Программная LLM (`core/llm.py`) — только для автономного пути.

**Подключение к Claude Code.** Через `.mcp.json` (запускает `scripts/mcp-server.sh`). См. `.claude/docs/setup.md`.
**Статус:** ✅ реализовано. Смоук-тесты — `tests/test_mcp_server.py`.

**Инварианты / подводные камни.**
- ⚠️ Read-only по Telegram: сервер НЕ объявляет инструментов отправки сообщений.
- Секреты и файл сессии наружу не отдаются (например, `get_profile` отдаёт cv.md без контактов-секретов по политике).
- Долгие операции (сканирование архива) — стримить/чанковать, помнить про FloodWait.
