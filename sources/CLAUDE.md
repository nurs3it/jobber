# sources/ — источники вакансий (реализации JobSource)

**Назначение.** Конкретные провайдеры. Каждый реализует `core.interfaces.JobSource` и маппит
свои данные в доменные модели `core/`.

**Подпапки.**
- `telegram/` — полная реализация (Telethon, user session). Единственный рабочий источник сейчас.
- `linkedin/` — заглушка (Фаза 7). Учесть ToS и официальный API.
- `webboards/` — заглушка (Фаза 7). Предпочитать официальные API/RSS.

**Как добавить источник.** См. `.claude/docs/adding-a-source.md`: реализовать интерфейс,
выставить честные capability-флаги, написать маппинг, зарегистрировать в `registry.py`,
добавить секцию в `config.yaml`.

**Инварианты / подводные камни.**
- ⚠️ Все источники — read-only по отношению к платформе. `can_apply=False`.
- Источник владеет смыслом своего `cursor` (ядро не интерпретирует его).
- Телеграм-специфика (Telethon-объекты) не должна протекать в `core/` — только модели.
