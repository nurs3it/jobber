# /onboard — настройка за один раз

Цель: после этой команды у пользователя есть `profile/cv.md`, заполненный `.env` и рабочая сессия Telegram.
Действуй пошагово, дружелюбно, по-русски. Не угадывай молча — при нехватке данных спрашивай.

## Шаг 1. Резюме → profile/cv.md

1. Попроси пользователя приложить резюме файлом: **pdf, docx или изображение** (png/jpg).
2. Извлеки текст:
   - **PDF / DOCX / TXT** → запусти CLI: `jobber extract <путь>`. Команда печатает извлечённый текст
     в stdout (pdf→pypdf+fallback pdfplumber; docx→параграфы+таблицы).
   - **Изображение** → прочитай его напрямую **зрением Claude** (лучший путь). Программный
     fallback — `jobber extract image.png` (OCR pytesseract, нужен системный tesseract).
3. **Структурируй в `cv.md` по шаблону `profile/cv.md.template`** и запиши в `~/.jobber/profile/cv.md`
   (Write tool). Это источник истины для скоринга. Покажи итог пользователю.

### ⚠️ ГЛАВНОЕ ПРАВИЛО АНАЛИЗА РЕЗЮМЕ — НИЧЕГО НЕ УПУСКАТЬ

Резюме у всех разные по структуре, языку и полноте. Твоя задача — перенести в `cv.md`
**все факты исходника без потерь**, нормализовав их в секции шаблона. Конкретно:

- **Полнота прежде красоты.** Лучше сохранить лишнюю деталь, чем потерять. Каждый пункт опыта,
  каждый навык, проект, сертификат, ссылка, цифра достижения — должны попасть в cv.md.
- **Не выдумывай.** Не добавляй навыки/опыт, которых нет в резюме. Если поле отсутствует —
  ставь честную пометку «не указано», а не догадку.
- **Нормализуй, но сохраняй исходное.** Грейд (junior/middle/senior/lead), годы опыта,
  зарплатную вилку, локацию, формат (удалёнка/гибрид/офис), готовность к релокации —
  приведи к единому виду в секции «Условия» и «Желаемая роль», но не теряй формулировки резюме.
- **Раскладывай по секциям шаблона:** Кратко, Желаемая роль (роль/грейд/ключевые слова),
  Навыки, Опыт работы, Образование, Языки, Условия, Стоп-факторы, Контакты, Прочее.
  Всё, что не вписалось в явные секции, помещай в «Прочее» — но не выбрасывай.
- **Ключевые слова для матчинга.** В «Желаемая роль → Ключевые слова» собери технологии/роли,
  по которым реально искать вакансии (например: React, TypeScript, Vue, Node, Frontend).
- **Стоп-факторы.** Если в резюме/словах пользователя есть «не рассматриваю X» — занеси в «Стоп-факторы».
- **Язык.** Пиши cv.md на языке резюме (обычно RU). Технические термины оставляй как есть.
- **Сохрани контакты** пользователя (Telegram/email/телефон) в секцию «Контакты» — они нужны,
  чтобы пользователь вставлял их в отклик вручную.

После черновика cv.md **покажи его пользователю и спроси, всё ли верно** (особенно роль, грейд,
зарплатные ожидания, локацию, удалёнку, стоп-факторы). Правки вноси сразу. Запиши финал в `profile/cv.md`.

### Заполнение config.yaml из резюме

Перенеси извлечённые фильтры в `config.yaml → filters` (roles, location, remote_only,
relocation_ok, salary_min/currency, stop_factors), подтвердив значения у пользователя.

## Шаг 2. Telegram-ключи → .env

1. Файл `~/.jobber/.env` создаётся при `jobber setup`. Если его нет — попроси выполнить `jobber setup`.
2. Объясни, где взять `api_id`/`api_hash`: https://my.telegram.org → API development tools.
   Если ключей ещё нет — помоги получить.
3. Запиши `TG_API_ID`, `TG_API_HASH`, `TG_PHONE` в `~/.jobber/.env` (через Edit tool по строкам, чтобы
   не светить секреты в argv/логах). **Никогда не повторяй секреты в чат и не логируй их.**

## Шаг 3. Глубина первого прохода

Спроси, за сколько дней сканировать историю/архив при первом запуске (предложи 7 / 14 / 30).
Запиши в `config.yaml → first_pass.depth_days`. Уточни время ежедневного дайджеста (`digest.schedule_time`).

## Шаг 4. Первый логин в Telegram

1. Запусти интерактивный логин: `jobber login`. Telethon (user session) запросит код и пароль 2FA.
2. Пользователь вводит код из Telegram; при включённой 2FA — пароль. Интерактивно в терминале.
3. Создастся файл сессии `<TG_SESSION_NAME>.session`. 🔒 Напомни: это полный доступ к аккаунту,
   он локальный и в .gitignore. Никогда не коммитить и не пересылать.

## Готово

Подтверди: cv.md заполнен, ключи сохранены, сессия активна. Подскажи следующий шаг — `/scan`.
