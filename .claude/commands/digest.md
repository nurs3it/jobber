# /digest — дайджест в Obsidian

Собрать новые подходящие вакансии и записать датированный Markdown для Obsidian.
LLM-шаги (классификация, скоринг, письма) делаешь ТЫ сам (Claude Code) по `cv.md`.

## Шаги

1. **Профиль и кандидаты.** Как в `/scan`: `get_profile()` + `scan_candidates(depth_days=$ARGUMENTS,
   include_archived=true)` (MCP).
2. **Классифицируй и оцени** каждого кандидата (вакансия? поля? контакт? скор 0–100 + причина).
   Оставь score ≥ `scoring.min_score`.
3. **Сопроводительные.** Для вакансий со скором ≥ `scoring.cover_letter_threshold` напиши короткий
   персональный черновик письма под конкретную вакансию + `cv.md`. Тон/язык — из `config.cover_letter`
   (по умолчанию дружелюбный, молодёжный, RU; для англоязычной вакансии — EN). Только черновик.
4. **Запись.** Передай список в `write_digest(items, date?)` (MCP). Сервер сделает дедуп, отсортирует
   по скору и запишет файл `vault/Jobber/ГГГГ-ММ-ДД-jobber.md` с карточками (ссылка · кому писать ·
   сопроводительное · скор+причина · источник · дата). Если новых нет — короткая пометка, не пустой файл.

   Формат каждого элемента `items`:
   `{title, company, link, location, remote, salary, score, reason, cover_letter,
     dialog_title, dialog_id, message_id, text, contact:{found, username, url, bot, raw, note}}`.
5. **Отчёт в чат.** Продублируй краткую сводку и путь к файлу дайджеста.

> ⚠️ Не печатать в дайджест секреты, сессию или сырой CV. Письма — только черновики.
