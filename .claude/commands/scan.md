# /scan — разовый проход

Найти свежие вакансии в чатах, группах, каналах и сообществах пользователя, **включая архив**.
LLM-классификацию/скоринг делаешь ТЫ сам (Claude Code) по `cv.md` — отдельный API-ключ не нужен.

## Шаги

1. **Профиль.** Получи `get_profile()` (MCP) — это `cv.md`, источник истины для скоринга.
2. **Кандидаты.** Вызови `scan_candidates(depth_days=$ARGUMENTS, include_archived=true)` (MCP).
   Сервер переберёт все диалоги + архив, отсеет по предфильтру, обновит seen/курсоры и вернёт
   список сообщений-кандидатов (`dialog_title`, `text`, `link`, `date`, ...). Если число дней не задано —
   работает по курсорам (новое с прошлого раза); сервер сам ограничивает первый проход.
3. **Классификация и скоринг (сам Claude).** Для каждого кандидата реши:
   - это вакансия? (иначе пропусти);
   - извлеки `title`, `company`, `location`, `remote`, `salary`;
   - оцени релевантность профилю **0–100** с краткой причиной (учти навыки, грейд, локацию/удалёнку,
     стоп-факторы из `cv.md`/`config.yaml`). Бери только score ≥ `scoring.min_score`.
   - контакт уже частично виден в тексте (@username/ссылка/бот/почта) — зафиксируй «кому писать»
     или честно «контакт не указан».
4. **Отчёт в чат.** Кратко: просмотрено кандидатов / признано вакансиями / прошло порог. Покажи топ
   с скором и причиной. Предложи `/digest`, чтобы записать в Obsidian с письмами.

> Read-only. Никаких отправок. Письма генерируются на этапе `/digest` или по `/cover <id>`.
