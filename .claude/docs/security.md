# Безопасность — модель угроз

Jobber работает под **личным Telegram-аккаунтом** пользователя. Это требует осторожности.

## Активы

- **Файл сессии Telethon (`*.session`)** — 🔒 самый чувствительный актив. **= полный доступ к аккаунту.**
- **Секреты `.env`** — `TG_API_ID`, `TG_API_HASH`, `TG_PHONE`.
- **Резюме / `profile/cv.md`** — персональные данные.

## Правила

1. **Только локально.** Сессия, `.env`, CV — на машине пользователя. Всё в `.gitignore`.
2. **Никогда не коммитить и не логировать** сессию и секреты. Не выводить их в дайджест или чат.
3. **Read-only по Telegram.** Никаких отправок, вступлений, рассылок. MCP-сервер не объявляет
   инструментов отправки. Сопроводительные письма — только черновики; отклик делает пользователь руками.
4. **Мягкое поведение.** Задержки между запросами; `FloodWaitError` → ждать указанное время.
   Агрессивная активность может привести к ограничениям аккаунта со стороны Telegram.

## Угрозы и меры

| Угроза | Мера |
|---|---|
| Утечка `*.session` → угон аккаунта | gitignore, локальное хранение, обращаться как с паролем |
| Секреты в логах | политика «не логировать секреты», ревью логов |
| Блокировка/ограничение Telegram за автоматизацию | read-only, задержки, обработка FloodWait |
| Утечка персональных данных из CV | cv.md gitignored, не печатать в дайджест/чат лишнего |
| Случайная отправка сообщения | архитектурный запрет: нет инструментов отправки |

## Риск ограничений Telegram

User-session автоматизация — серая зона ToS Telegram. Снижаем риск: только чтение, человеческие
задержки, без массовых действий. Пользователь использует это на свой аккаунт и под свою ответственность.

## Будущие источники (ToS)

- **LinkedIn / веб-доски:** автоматизированный сбор ограничен их условиями. Предпочитать официальные
  API/RSS, соблюдать rate limits. Часть источников останется read-only. См. `adding-a-source.md`.

## Что нельзя

- Отправлять сообщения/отклики автоматически.
- Коммитить или пересылать сессию, `.env`, CV.
- Логировать секреты.
- Массово вступать/писать/скрейпить в обход задержек и ToS.
