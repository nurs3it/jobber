# Troubleshooting

## FloodWaitError (Telegram просит подождать)
- Причина: слишком частые запросы. Telethon бросает `FloodWaitError` с `e.seconds`.
- Решение: подождать указанное время (Jobber делает это автоматически). Увеличить
  `config.sources.telegram.delay_between_requests`. Не запускать несколько сканов параллельно.

## Протухшая / невалидная сессия
- Симптом: ошибки авторизации, просьба войти заново.
- Решение: повторить логин через `/onboard` (шаг 4). При необходимости удалить старый `*.session`
  и залогиниться заново (код + 2FA). 🔒 Не коммитить и не пересылать файл сессии.

## Плохой OCR резюме-картинки
- Симптом: в `cv.md` пропали или искажены данные из изображения.
- Решение: в Claude Code изображение читает зрение Claude — обычно этого достаточно.
  Если запускаете автономно — поставьте extra `ocr` (`pytesseract` + системный `tesseract`).
  Лучше всего прислать резюме в **pdf/docx** — текст извлекается точно. Всегда проверяйте cv.md
  на полноту после импорта (правило «ничего не упускать»).

## Дубликаты вакансий в дайджесте
- Причина: одна вакансия репостнута в разных каналах.
- Решение: fuzzy-дедуп управляется `config.digest.dedup_similarity` (выше = строже).
  Точные повторы режет seen-store. Если дублей много — понизить порог.

## Пустой дайджест
- Это нормально, если новых вакансий нет — будет файл с короткой пометкой, не пустой.
- Если ожидали вакансии: проверьте период (`/scan 30`), `exclude_dialogs`, `scoring.min_score`
  (слишком высокий порог отсекает), ключевые слова предфильтра, актуальность `cv.md`.

## Не находит вакансии вообще
- Проверьте, что сессия активна и `iter_dialogs` видит диалоги (вкл. архив).
- Проверьте предфильтр: возможно, нужно добавить ключевые слова в `config.prefilter`.
- `prefilter.enabled: false` временно пропустит всё в LLM (дороже) — для диагностики.

## Сброс состояния
- `/seen-reset` очищает курсоры и seen-store; следующий `/scan` пройдёт заново (по `first_pass.depth_days`).
